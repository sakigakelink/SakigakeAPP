## 1. 概要：初期配置・ハード制約・ソフト制約

ソルバーの制約は **初期配置**、**ハード制約**、**ソフト制約** の3層で構成される。

| 種類 | 意味 | 違反時の挙動 |
|------|------|-------------|
| **初期配置** | ソルバー実行前に確定しているシフト割当。ソルバーはこれを動かさない | 変更不可（ハード制約として実装） |
| **ハード制約** | 絶対に守らなければならないルール | 違反する解は生成されない（infeasible） |
| **ソフト制約** | 守ることが望ましいルール | 違反可能だがペナルティが加算され、ソルバーが最小化を目指す |

ソルバーはまず初期配置を固定し、残りのマスについて全てのハード制約を満たす解の中から、ソフト制約のペナルティ合計が最小の解を探す。ハード制約を1つでも満たせない場合、解なし（infeasible）となりエラーを返す。

> **有給（paid）は初期配置に含まない。** 労基法上、有休は使用者が定めたシフト表に対して労働者が後から請求する権利であり、概念的に初期配置とは異なる。ソルバーは希望日にのみ配置するが、これはハード制約（H6）として扱う。

## 2. 勤務種別・勤務区分

### 2.1 シフト種別

| コード | 名称 | 説明 |
|--------|------|------|
| `day` | 日勤 | 標準的な日中の勤務 |
| `late` | 遅出 | 遅い時間帯の勤務 |
| `night2` | 夜勤（二交代） | 16時間の夜間勤務 |
| `junnya` | 準夜（三交代） | 三交代制の準夜勤務 |
| `shinya` | 深夜（三交代） | 三交代制の深夜勤務 |
| `ake` | 明け | 二交代夜勤翌日の回復日 |
| `off` | 公休 | 通常の休日 |
| `paid` | 有給 | 有給休暇 |
| `refresh` | リフ休 | リフレッシュ休暇 |

### 2.2 勤務区分（ワークタイプ）

| 区分コード | 区分名 | 使用可能な勤務 | ソルバー対象 |
|------------|--------|----------------|--------------|
| `2kohtai` | 二交代 | day, late, night2, ake, off, paid, refresh | 対象 |
| `3kohtai` | 三交代 | day, late, junnya, shinya, off, paid, refresh | 対象 |
| `day_only` | 日勤のみ | day, off, paid, refresh | 対象 |
| `night_only` | 夜勤専従 | night2, ake, off, paid, refresh | 対象 |
| `fixed` | 固定パターン | パターンに依存 | **対象外** |

区分外の勤務は割当不可（ハード制約、forbidden_mapで実装）。

### 2.3 祝日の扱い

- **データソース**: `holidays.json` に定義
- **扱い**: 土日と同等（休日用の人員配置を適用）
- **公平性**: 土日勤務カウントに祝日出勤も含む

## 3. ハード制約一覧

以下は全て **絶対に緩和してはならない** 制約。違反する解は生成されない。

### 3.1 全病棟共通

| # | 制約 | 説明 |
|---|------|------|
| H1 | **1日1シフト** | 各職員は1日に1つのシフトのみ |
| H3 | **月間公休日数** | `off` の日数 == `monthlyOff`（2月=8日、5月=10日、他=9日） |
| H4 | **7連勤禁止** | 最大6日連続勤務。off/paid/ake/refresh で連続リセット |
| H5 | **maxNight上限** | 職員ごとの月間夜勤上限回数 |
| H6 | **有給は希望日のみ** | paid は本人の assign 希望がある日のみ割当可 |

### 3.2 初期配置（ソルバー実行前に確定するシフト）

初期配置はソルバーが動かさない確定済みの制約。全てハード制約として実装されている。

| 種類 | 内部値 | 説明 |
|------|--------|------|
| **指定（assign）** | `type:"assign"` | 特定日に特定シフトを強制。例: 「田中の5日をoff」 |
| **除外（avoid）** | `type:"avoid"` | 特定日の特定シフトを禁止。例: 「佐藤の3日にjunnyaを入れない」 |
| **リフレッシュ** | `shift:"refresh"` | 本人のassign希望がある日にのみ配置（指定の一種） |
| **前月末制約** | `forced_by_prev` | 前月末の勤務に基づく自動強制（下記参照） |

#### H2: 希望シフト厳守

全ての指定（assign）・除外（avoid）希望はハード制約。**いかなる理由があっても緩和不可**。
- 指定: `model.Add(shifts[職員, 日, シフト] == 1)`
- 除外: `model.Add(shifts[職員, 日, シフト] == 0)`

#### H7: リフレッシュは希望日のみ

refresh は本人の assign 希望がある日にのみ割当可。

#### 前月データの引き継ぎ

月をまたぐ際に以下のデータが自動的に引き継がれ、初期配置としてハード制約に反映される:

| データ | 用途 |
|--------|------|
| 最終日の勤務種別 | night2→ake→off シーケンスの継続、junnya翌日制限 |
| 連続勤務日数 | 7連勤禁止の月境界判定 |

> maxNight は月単位でリセットされる（毎月独立して0から計算）。

### 3.3 人員配置

| # | 制約 | 説明 |
|---|------|------|
| H8 | **準夜帯人数** | junnya + night2 == reqJunnya（毎日厳守、曜日不問） |
| H9 | **深夜帯人数** | shinya + ake == reqShinya（毎日厳守、曜日不問） |
| H10 | **遅出人数** | late == reqLate（対応病棟のみ） |
| H11 | **日祝日勤上限** | 日祝の日勤人数 <= reqDayHoliday + 1（過剰配置防止） |

> 日勤帯の**下限**はソフト制約（§4参照）。準夜・深夜帯は完全一致のハード制約。

### 3.4 二交代制

| # | 制約 | 説明 |
|---|------|------|
| H12 | **night2→ake強制** | night2の翌日は必ずake |
| H13 | **ake→休み強制** | akeの翌日は必ずoff/paid/refresh |
| H14 | **禁止勤務** | junnya, shinya は割当不可 |

月境界: 前月末night2 → 当月1日ake強制。前月末ake → 当月1日休み強制。

### 3.5 三交代制

| # | 制約 | 説明 |
|---|------|------|
| H15 | **準夜翌日制限** | junnnyaの翌日は junnya または off/paid/refresh のみ |
| H16 | **3連続夜勤禁止** | junnya/shinyaは最大2連続まで |
| H17 | **禁止勤務** | night2, ake は割当不可 |

許可される2連続パターン: shinya→shinya、junnya→junnya、shinya→junnya。
junnya→shinyaはH15により不可。

月境界: 前月末junnya → 当月1日はjunnya/off/paid/refreshのみ。

### 3.6 一病棟の追加制約

| # | 制約 | 設定元 |
|---|------|--------|
| H18 | **遅出完全禁止** | config.json `shiftRestrictions.late: false` |
| H19 | **有資格者（nurse+junkango）日勤最低人数** | 曜日別: 火土=3名、水金=4名、日祝=2名、月木=制限なし |
| H20 | **全帯nurse最低1名** | 日勤・準夜帯・深夜帯それぞれにnurse（正看護師）が最低1名 |
| H21 | **看護補助者の夜勤上限** | 準夜帯: nurseaide最大1名/日、深夜帯: nurseaide最大1名/日 |

### 3.7 スキルポイント制約（全病棟共通フレームワーク、config駆動）

| # | 制約 | 設定元 |
|---|------|--------|
| H22 | **夜勤帯の新人重複禁止** | 準夜帯・深夜帯それぞれにskillPoint=0の職員が2名以上入らない。config.json `noDoubleRookie.enabled` |

> 日勤帯は対象外（日勤帯のスキルレベルは§4のソフト制約で管理）。

各職員の `skillPoint`（0〜5の整数）は `employees.json` に設定。0=新人、5=ベテラン。

## 4. ソフト制約（ペナルティ）一覧

ソルバーはペナルティ合計を最小化する。違反しても解は生成されるが、低品質になる。

### 4.1 人員配置

| 制約 | ペナルティ | 説明 |
|------|-----------|------|
| 日勤人数不足 | **500点/人/日** | 曜日別目標（平日: reqDayWeekday、日祝: reqDayHoliday）の未達 |
| スキルポイント合計不足 | **400点/不足SP/帯/日** | 各時間帯のスキルポイント合計が曜日別最低値に未達。config.json `skillPointMinimumPerBand` で帯別・曜日別に設定 |

#### 曜日別日勤人数調整

一病棟・二病棟は曜日別に日勤目標を調整する（config.json `dayAdjustment`）。

**一病棟**:

| 曜日 | 調整 |
|------|------|
| 月・木 | 基準値 |
| 水・金 | 基準値−1 |
| 火・土 | 基準値−2 |
| 日・祝 | 休日基準値（上限あり） |

**二病棟**:

| 曜日 | 調整 |
|------|------|
| 月・金 | 基準値 |
| 木 | 基準値+1（処置・検査多い） |
| 火・水・土 | 基準値−1 |
| 日・祝 | 休日基準値（上限あり） |

### 4.2 夜勤公平性

| 制約 | ペナルティ | 説明 |
|------|-----------|------|
| 夜勤レンジ（段階） | 差1: **200点**、差2: **500点**、差3: **800点**、差4: **1000点**、差5+: **1500点** | 職員間の夜勤回数差（正規化済み）。段階的に累積 |
| 週末勤務偏り | **diff × 40点** | 週末勤務回数の max-min 差 |
| 遅出回数偏り | **diff × 20点** | 遅出回数の max-min 差（late有効な病棟のみ） |

### 4.3 連勤

| 制約 | ペナルティ | 説明 |
|------|-----------|------|
| 5日連続勤務 | **150点/件** | |
| 6日連続勤務 | **300点/件** | ハード上限は7連勤禁止 |

### 4.4 二交代のローテーション

| 制約 | ペナルティ | 説明 |
|------|-----------|------|
| 夜勤間隔2日 | **30点/回** | night2が2日間隔で連続 |
| 夜勤間隔3日 | **10点/回** | night2が3日間隔 |

### 4.5 三交代のローテーション

#### 基本ペナルティ

| 制約 | ペナルティ | 説明 |
|------|-----------|------|
| 連続夜勤 | **20点/回** | junnya/shinya連続（junnya→junnnyaは除外） |
| 日勤帯→深夜 | **25点/回** | day/late(d) → shinya(d+1)。インターバル約7時間 |
| 準夜・深夜バランス | **diff × 10点** | 個人の月間 junnya-shinya 回数差 |
| 準夜・深夜半月偏り | **diff × 15点** | 個人の前半/後半 junnya-shinya 差 |

#### リフレッシュ日数で変動するペナルティ

リフレッシュ休暇を取得する職員は通常より多くの休みを得るため、きつめのローテーションを許容する。

| 制約 | 0日 | 1日 | 2日 | 3日以上 | 説明 |
|------|-----|-----|-----|---------|------|
| 深夜後非休 | **20点** | 12点 | 7点 | 3点 | shinya翌日が休みでも深夜連続でもない |
| 散発夜勤 | **15点** | 10点 | 6点 | 3点 | shinya→休→shinya パターン |
| 準夜→休→深夜 | **50点** | 35点 | 25点 | 12点 | 体内リズム切替負担 |
| 好ましいローテーション | **-15点** | -10点 | -6点 | -3点 | shinya→休→junnya、shinya→junnya→休（ボーナス） |

> ボーナスは目的関数から減算されるため、ソルバーがこれらのパターンを積極的に採用する。

#### 希望休み前後の夜勤配慮

> **重要**: 以下の「希望休み」にはリフレッシュ休暇（refresh）も含む。

| 制約 | ペナルティ | 説明 |
|------|-----------|------|
| 準夜→希望休→深夜 | 上記 **+50点** 上乗せ | 準夜→休→深夜ペナルティに加算 |
| 希望休み前日が準夜 | **10点/回** | せっかくの休みの前夜が遅番勤務 |
| 希望休み翌日が深夜 | **10点/回** | 休み明けにいきなり深夜勤務 |

> 希望休み前後ペナルティ（10点）はリフレッシュ日数に関わらず固定。

### 4.6 デフォルト人員配置（参考）

| 病棟 | reqDayWeekday | reqDayHoliday | reqJunnya | reqShinya | reqLate | maxLate |
|------|:---:|:---:|:---:|:---:|:---:|:---:|
| 1病棟 | 7 | 5 | 2 | 2 | 0 | 0 |
| 2病棟 | 7 | 5 | 2 | 2 | 1 | 4 |
| 3病棟 | 7 | 5 | 3 | 2 | 0 | 0 |

## 5. ソルバーの最適化戦略

### 解法モード

| モード | 制限時間 | 用途 |
|--------|---------|------|
| `quick` | **15秒** | 高速な仮スケジュール |
| `balanced` | **45秒** | 通常使用 |
| `quality` | **90秒** | 最高品質 |

### フォールバック戦略

厳密解法（attempt_level=1）のみ実行。解が見つからない場合はエラーを返す（制約緩和なし）。

> `_solve_core()` には L2〜L5+ の緩和コードが残存しているが、`solve()` は常に `attempt_level=1` で呼び出すためデッドコード。

### 目的関数（最小化対象）

```
最小化:
  人員不足ペナルティ（日勤500点/人 + minNight不足300点 + 1病棟職種制約）
  + 夜勤レンジ段階ペナルティ               ... 夜勤公平性（差が大きいほど急増）
  + 週末勤務偏り × 40                     ... 週末公平性
  + 遅出偏り × 20                         ... 遅出公平性
  + 夜勤間隔ペナルティ                     ... 夜勤の間隔（二交代・三交代それぞれ）
  + 三交代連続夜勤 × 20                   ... 連続夜勤抑制
  + 準夜・深夜バランスペナルティ            ... 三交代バランス
  + 散発夜勤ペナルティ                     ... 夜勤の散発防止
  + 深夜後非休ペナルティ                   ... 深夜後の休息
  + 日勤帯→深夜ペナルティ × 25           ... 三交代専用（インターバル約7時間）
  + 準夜→休→深夜ペナルティ               ... 体内リズム切替負担
  + 希望休み前後夜勤ペナルティ             ... 希望休みの質の確保
  - 好ましいローテーションボーナス         ... 深夜→休→準夜、深夜→準夜→休を優遇
  + 連続勤務ペナルティ                     ... 5連勤150点・6連勤300点
```

各ペナルティには重みグループ（`total_violation`, `night_range`, `consecutive`, `weekend`, `rotation`）があり、`weight_profile` で調整可能（デフォルトは全て1）。

## 6. 品質スコアと評価基準

### スコア配分（100点満点）

| カテゴリ | 配点 | 評価対象 |
|----------|------|---------|
| 夜勤の公平性 | **25点** | スタッフ間の夜勤回数バランス |
| 連続勤務 | **20点** | 過度な連続勤務の抑制 |
| シフトパターンの規則性 | **20点** | 勤務リズムの安定性 |
| 土日の公平性 | **15点** | 休日出勤の均等配分 |
| 希望の充足率 | **20点** | 職員希望の反映度 |

### 評価ランク

| スコア | ランク | 評価 |
|--------|--------|------|
| 90〜100 | ★★★★★ | 優秀 |
| 80〜89 | ★★★★☆ | 良好 |
| 70〜79 | ★★★☆☆ | 普通 |
| 60〜69 | ★★☆☆☆ | やや不足 |
| 0〜59 | ★☆☆☆☆ | 要改善 |

### 公平性の測定

| 指標 | 算出方法 | 基準 |
|------|---------|------|
| ジニ係数 | 0（完全平等）〜 1（完全不平等） | < 0.1: 優秀 / 0.1-0.2: 良好 / 0.2-0.3: 普通 / > 0.3: 要改善 |
| 変動係数（CV） | 標準偏差 ÷ 平均 | 低いほど良い |
| 夜勤レンジ | 最大回数 − 最小回数 | 0に近いほど良い |

> **注**: 本ドキュメントはソースコード（`solver.py`, `engines/*/config.json` 等）に基づく。設定値は病棟設定（`ward_settings.json`）により変更可能。
