# SakigakeAPP 開発手順書

## この手順書について
Claude Code を使って SakigakeAPP を開発・更新するための手順書です。
gitの知識がなくても、この手順に沿って指示を出せば安全に運用できます。

---

## 全体の流れ（3ステップ）

```
[1] ローカルで編集  →  [2] GitHubに保存  →  [3] 本番サーバーに反映
  (あなたのPC)         (クラウド保管)        (mining-base)
```

---

## 基本的な指示の出し方

### コードを修正したいとき

Claude Codeの新しいセッションを開いて、以下のように指示します：

```
SakigakeAPP の〇〇を修正してください。
修正後、テストを実行して問題なければコミット＆プッシュしてください。
リモート（mining-base）にも反映してください。
```

これだけで Claude が以下を自動で行います：
1. コードの修正
2. テストの実行（問題がないか確認）
3. GitHubへの保存（コミット＆プッシュ）
4. 本番サーバーへの反映（git pull）

### 動作確認だけしたいとき

```
mining-base でシフトのテストを実行してください。
```

### 現在の状態を確認したいとき

```
SakigakeAPP の現在のgit状態を確認してください。
ローカルとリモートが同期しているか教えてください。
```

## 指示のテンプレート集

### バグ修正
```
シフトの solver.py で〇〇のバグがあります。修正してください。
テスト実行 → コミット → プッシュ → リモート反映 まで行ってください。
```

### 機能追加
```
シフトに〇〇の機能を追加してください。
計画を立ててから実装してください。
テスト実行 → コミット → プッシュ → リモート反映 まで行ってください。
```

### 設定変更
```
シフトの〇〇の設定を△△に変更してください。
コミット → プッシュ → リモート反映 まで行ってください。
```

---

## 重要なルール

### やっていいこと
- Claude Code に修正を依頼する
- 修正内容の説明を求める
- テスト結果の確認を求める

### やらないほうがいいこと
- リモート（mining-base）上で直接ファイルを編集する
  → 必ずローカルで編集してgit経由で反映する
- git の履歴を手動で操作する（reset, rebase など）
  → 必要があれば Claude に依頼する

---

## トラブル時の対応

### 「コンフリクト」と言われたら
```
コンフリクトを解消してください。現在の変更内容を教えてください。
```

### 「テストが失敗した」と言われたら
```
失敗したテストの内容を教えてください。修正してからやり直してください。
```

### リモートとローカルがずれていると言われたら
```
ローカルとリモートの状態を確認し、安全に同期してください。
```
