# 処方入力アプリケーション — 言語的要件定義書

## 1. 概要

精神科病院における入院・外来処方の入力支援Webアプリケーション。
薬剤の選択・用量設定・用法指定をGUIで行い、処方箋テキストを自動成形する。

**技術スタック**: React + TypeScript + Vite（SPA、外部UIライブラリなし）

---

## 2. 設計思想

本アプリケーションは「処方を入力できる」だけでなく、精神科病院の実務に即した以下の設計方針を持つ。

### 2.1 添付文書準拠の自動用法設定 — 選択ミスの構造的排除

**課題**: 既存の処方入力システムでは、薬剤を選択した後に毎回手動で用法を選ぶ。睡眠薬を「毎食後」で誤入力する、ラツーダを空腹時に処方してしまう等のヒューマンエラーが起こりうる。

**設計**: 添付文書で服用タイミングが規定されている薬剤（睡眠薬→寝る前、インヴェガ→朝食後、ラツーダ→夕食後 等）は、薬剤を選択した時点で用法が自動的にセットされる。手動での変更も可能だが、デフォルトが正しいことで「何も考えずに進めても安全」な状態を作る。

### 2.2 一包化サマリーの自動集計 — 調剤業務の直接支援

**課題**: 入院定期処方は全て一包化（朝・昼・夕・寝る前ごとにまとめて分包）される。従来、薬剤師はRpごとの処方箋から各スロットの錠数を手計算で集計しており、処方変更のたびに再計算が必要だった。

**設計**: 全ての定期処方・不均等処方からスロット別の総錠数を自動集計し、「朝3 昼1 夕4（計8錠/日）」として処方箋末尾に表示する。頓用は一包化対象外のため自動で除外する。処方内容を変更すれば集計も即座に更新されるため、調剤側の確認作業を大幅に削減する。

### 2.3 不均等投与の専用UI — フリーテキスト依存からの脱却

**課題**: 不均等投与（朝1錠・夕2錠 等）は臨床上よくあるパターンだが、多くの処方システムでは備考欄にフリーテキストで記載するか、スロットごとに別々のRpとして手動入力する。記載の揺れや転記ミスが起きやすい。

**設計**: 4スロット（朝食後・昼食後・夕食後・寝る前）に錠数を直接入力する専用UIを用意した。用法ラベル（例:「朝夕食後(朝1-夕2)」）は入力内容から自動生成される。成形時は1つのRp内で不均等表記（「1日3T（朝1T・夕2T）」）にまとめられ、処方箋の規格に準拠する。人間がやるのは「どのスロットに何錠」を指定するだけである。

### 2.4 リアルタイム薬価表示 — 処方時のコスト意識

**課題**: 薬価は処方後に医事課や薬局で計算されるのが一般的で、処方医がコストを意識する機会が少ない。後発品と先発品の価格差、規格の違いによるコスト差が処方時に見えない。

**設計**: 規格選択ボタンに薬価を併記し、Rp単位および合計の1日薬価を即座に表示する。頓用は使用頻度が不定のため合計から除外し「(屯用除く)」と明示することで、定期処方の正確なランニングコストが一目でわかる。

### 2.5 最大量超過の警告付き許容 — 精神科の臨床判断を尊重

**課題**: 処方システムの最大量チェックは一般にブロック型（入力不可）か無視型（警告なし）の二択になりがちである。しかし精神科では、治療抵抗性の患者に対して添付文書上の最大量を超えて投与する臨床判断が正当に存在する。

**設計**: 1日最大量を超過した場合は明確な警告を表示するが、「⚠ 最大量超過で追加」ボタンで追加を許容する。ブロックせず注意喚起にとどめることで、医師の裁量を妨げずに安全網を提供する。

### 2.6 割り切れない錠数の即時可視化 — 処方ミスの早期発見

**課題**: 「3錠を朝夕2回で」と指定すると1回1.5錠になるが、割線のない錠剤では実現不可能である。これは処方箋を出してから薬局で初めて気づくことが多い。

**設計**: 定期モードで錠数÷回数が割り切れない場合、入力時点で即座に「3T÷2回=1.50T/回（割り切れません）」と警告を表示する。処方箋の成形前に気づけることで、疑義照会の手間と処方修正の遅延を防ぐ。

## 4. 処方区分

### 4.1 入院処方
- **開始日**（日付）と**開始タイミング**（朝／昼／夕／寝る前）を指定する
- 終了日は指定しない（入院中は継続）

### 4.2 外来・臨時処方
- **開始日＋開始タイミング**と**終了日＋終了タイミング**の両方を指定する
- 開始タイミングのデフォルトは「朝」、終了タイミングのデフォルトは「夕」

## 5. 薬剤マスタ

### 5.1 構造
各薬剤は以下の情報を持つ：

| フィールド | 説明 |
|-----------|------|
| 商品名 | 院内で使用する名称 |
| 一般名 | 一般的薬剤名（検索用） |
| 薬効分類 | カテゴリ（抗精神病薬、睡眠薬、消化器系薬 等） |
| 規格 | 利用可能な規格値と薬価のペア（1〜4種類） |
| 単位 | 規格の単位（mg, μg, mg/包, mg/g, mg/mL） |
| 剤型 | 錠剤(T)／カプセル(C)／散剤(g)／液剤(mL) |
| 1日最大量 | 添付文書ベースの上限（任意） |
| デフォルト用法 | 添付文書で服用タイミングが規定されている場合の自動設定用法（任意） |

### 5.2 薬効分類（16カテゴリ）
抗精神病薬（非定型）、抗精神病薬（定型）、抗うつ薬、睡眠薬、抗不安薬、
気分安定薬・抗てんかん薬、抗パーキンソン薬、ADHD治療薬、認知症治療薬、
消化器系薬、循環器系薬、糖尿病薬、漢方薬、抗菌薬、鎮痛薬・解熱薬、その他

### 5.3 デフォルト用法の適用例
| 薬剤カテゴリ | デフォルト用法 | 理由 |
|-------------|---------------|------|
| 睡眠薬（全9種） | 寝る前 | 就寝前服用が添付文書で規定 |
| インヴェガ錠 | 朝食後 | 1日1回朝食後が規定 |
| レキサルティOD錠 | 朝食後 | 1日1回朝食後が規定 |
| ラツーダ錠 | 夕食後 | 食後服用が必須（吸収に影響） |

## 6. 用法マスタ

### 6.1 用法プリセット（57種）
用法を以下のカテゴリに分類し、プリセットとして提供する：

| カテゴリ | 例 | プリセット数 |
|---------|-----|------------|
| 食後 | 毎食後、朝夕食後、朝食後＋寝る前 等 | 11 |
| 食前 | 毎食前、朝食前、朝夕食前 等 | 6 |
| 食直前 | 毎食直前、朝食直前 等 | 5 |
| 食間 | 毎食間、朝夕食間 等 | 5 |
| その他 | 寝る前、起床時 | 2 |
| 隔日 | 朝食後 隔日、寝る前 隔日 等 | 6 |
| 週1回 | 朝食後 週1回、起床時 週1回 等 | 5 |

### 6.2 クイック選択
使用頻度の高い以下7種をクイック選択ボタンとして常時表示する：
- 毎食後、朝夕食後、朝食後、夕食後、毎食後＋寝る前、毎食前、寝る前

### 6.3 検索機能
クイック選択にない用法は「その他の用法を検索」から、ラベル名またはカテゴリ名で検索できる。

## 7. 用法モード（4モード）

### 7.1 定期モード
- 用法マスタからプリセットを選択する標準的な入力方式
- 処方の約95%がこのモードで入力される
- 同一用法の薬剤は1つのRpにグルーピングされる

### 7.2 頓用モード
- 「不穏時」「不眠時」等の条件を選択、または自由入力する
- プリセット条件: 不穏時、不眠時、疼痛時、発熱時、便秘時、嘔気時、不安時、頭痛時
- 1回量を指定（1日量ではない）
- 1日最大量チェックは行わない
- 各薬剤が独立した1Rpとなる

### 7.3 不均等モード
- 朝食後／昼食後／夕食後／寝る前の4スロットに個別の錠数を指定する
- 用法ラベルは入力内容から自動生成される（例:「朝夕食後(朝1-夕2)」）
- 成形時は1Rpにまとめて不均等表記する（「1日3T（朝1T・夕2T）」）。スロット分割はしない

### 7.4 体重換算モード
- 体重（kg）と用量（mg/kg/日）を入力する
- 投与回数（分1〜分4）を選択し、対応するスロットをチェックボックスで指定する
- スロット数は投与回数と一致しなければならない（バリデーション）
- 1日総量と1回量が自動計算され表示される
- 実装は暫定（小児科医の監修後に詳細決定）

---

## 8. 薬剤入力フロー

### 8.1 薬剤検索
1. テキスト入力（2文字以上）でリアルタイム検索が開始される
2. 商品名・一般名の両方が検索対象
3. 検索結果は最大15件のドロップダウンで表示される
4. 入力欄外をクリックするとドロップダウンが閉じる

### 8.2 薬剤選択時の自動処理
- 薬剤にデフォルト用法（`defaultUsageId`）が設定されている場合、用法が自動的に切り替わる
- デフォルト用法がない場合は「毎食後」が初期値となる

### 8.3 規格選択
- 薬剤に存在する全規格をボタンで表示する
- 各ボタンには規格値と薬価を併記する
- 体重換算モードでは規格選択を非表示にする

### 8.4 錠数選択
- −/+ボタンで1単位ずつ増減する（最小値1）
- 定期モード: 1日総量として入力（1回量は錠数÷回数で自動導出）
- 頓用モード: 1回量として入力
- 不均等モードでは非表示（スロット別入力で代替）
- 体重換算モードでは非表示
- 頓用モードでは「1回量」、定期モードでは「1日量」と表示する

### 8.5 用量警告
- **1日最大量超過**: 添付文書の上限を超えた場合に警告を表示する。追加は可能（「⚠ 最大量超過で追加」ボタン）
- **割り切れない錠数**: 定期モードで錠数÷回数が割り切れない場合に警告を表示する（例: 3T÷2回=1.50T/回）

### 8.6 追加条件
各モードで追加ボタンが有効になる条件：

| モード | 条件 |
|--------|------|
| 定期 | 薬剤選択済み、規格選択済み、錠数≧1、スロット≧1 |
| 頓用 | 薬剤選択済み、規格選択済み、錠数≧1、条件入力済み |
| 不均等 | 薬剤選択済み、規格選択済み、スロット合計≧1 |
| 体重換算 | 薬剤選択済み、用量>0、体重>0、回数>0、スロット数=回数 |

## 10. 処方箋成形

### 10.1 Rpグルーピング規則

成形処理は以下の2段階で行われる。

**第1段階: 同一薬剤統合（mergeSameDrugEntries）**

同一薬剤（drug.id）かつ同一規格（selectedStrength）のエントリが複数存在する場合、スロットを統合して1エントリにまとめる。入力方法（定期/不均等）に関わらず統合される。

- 例: デパケンR 200mg「朝食後 1T」＋「夕食後 2T」→ 朝夕食後（朝1-夕2）として1エントリに自動統合
- 統合後のスロット別用量が均等 → 定期（standard）エントリ
- 統合後のスロット別用量が不均等 → 不均等（custom）エントリ

**第2段階: 用法キーによるRpグルーピング**

| モード | グルーピング |
|--------|-------------|
| 定期 | 同一用法キー（スロット＋タイミング＋頻度）で1Rpにまとめる |
| 不均等 | アクティブスロットから用法キーを導出し、同一用法の定期薬と同一Rpにまとめる。不均等表記は薬剤行に付記（「1日3T（朝1T・夕2T）」） |
| 頓用 | 1薬剤＝1Rp（グルーピングしない） |
| 体重換算 | 1薬剤＝1Rp（暫定） |

**設計意図**: 外来処方では薬剤料の算定が「剤」（＝Rp）単位で行われるため（205円ルール）、同一用法の薬剤を不要に分割するとレセプト上の薬剤料が変わり得る。また内服薬の種類数が7以上で処方料減算の対象となるため、Rpの統合は算定上の実益がある。

### 10.2 用量構造（ResolvedDose）

内部データモデルとして、全モード共通の用量構造を持つ。

| フィールド | 意味 | 定期 | 不均等 | 頓用 | 体重換算 |
|-----------|------|------|--------|------|---------|
| perDose | 1回量 | 錠数÷回数 | 0（不均等） | 1回量 | 総量÷回数 |
| timesPerDay | 1日回数 | スロット数 | アクティブスロット数 | 0 | 分割回数 |
| dailyTotal | 1日総量 | 錠数 | スロット合計 | 0 | mg/kg×体重 |
| isUneven | 不均等 | false | true | false | false |
| slotDoses | スロット別 | ─ | 朝:1, 夕:2 等 | ─ | ─ |

この構造により、1回量・回数・1日量が常に明示的で、電子処方箋（HL7 FHIR MedicationRequest）への直接マッピングが可能。

### 10.3 出力テキスト形式

```
Rp1（薬価: XX.X円/日）
アリピプラゾールOD錠(6)2T
デパケンR錠(200) 1日3T（朝1T・夕2T）
2x(  )朝夕食後(朝1-夕2)

Rp2（薬価: XX.X円/回）
ワイパックス錠(0.5)1T
不穏時

─────────────────
合計薬価: XXX.X円/日(屯用除く)
一包化: 朝3 昼1 夕4（計8錠/日）
```

不均等の出力形式:
- 薬剤行: `薬名(規格) 1日X錠（朝XT・夕XT）`
- 用法行: `Nx(  )朝夕食後(朝X-夕X)`

### 10.3 薬価計算
- **定期処方**: 各Rpの1日薬価を合計する
- **頓用処方**: 1回あたりの薬価を表示する（合計薬価には含めない）
- 合計薬価に頓用が含まれない場合「(屯用除く)」を付記する

### 10.4 一包化サマリー
- 入院定期処方は全て一包化されるため、朝／昼／夕／寝る前ごとの総錠数を集計する
- 頓用は集計から除外する
- 合計薬価の下行に「一包化: 朝X 昼X 夕X 寝X（計X錠/日）」形式で表示する
- 値が0のスロットは省略する

### 10.5 一包化集計の計算方法

| モード | 計算方法 |
|--------|---------|
| 定期 | 錠数 ÷ スロット数 を各スロットに按分 |
| 不均等 | 各スロットの指定錠数をそのまま加算 |
| 体重換算 | 各スロットに1として暫定加算（TODO） |
| 頓用 | 集計対象外 |

### 10.6 剤型別の出力表記
- **錠剤・カプセル**: 「薬名(規格) X錠」
- **散剤・液剤**: 「薬名 X単位」（規格表記なし）

### 10.7 クリップボードコピー
成形結果テキスト全体をワンクリックでクリップボードにコピーできる。

## 11. 用法ラベル自動生成規則

### 11.1 定期モード
用法マスタのプリセットラベルをそのまま使用する。
プリセットにない場合はスロット＋タイミング＋頻度から自動生成する。

生成規則：
- 朝＋昼＋夕 → 「毎」を付与（例: 毎食後）
- 寝る前のみ → 「寝る前」
- 朝＋夕＋寝る前 → 「朝夕食後＋寝る前」
- 起床時 → 「起床時」
- 隔日・週1回 → 末尾に付記（例: 「朝食後 隔日」）

### 11.2 不均等モード
アクティブスロットから自動生成する。
- 食後固定（食事タイミングの選択なし）
- 例: 朝1・夕2 → 「朝夕食後(朝1-夕2)」

### 11.3 頓用モード
条件文字列をそのままラベルとする（例: 「不穏時」）。

### 11.4 体重換算モード
- 例: 「20mg/kg/日 朝昼夕食後（30kg）」
