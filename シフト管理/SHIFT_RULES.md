## 1. 勤務種別の定義

| コード | 名称 | 説明 |
|--------|------|------|
| `day` | 日勤 | 標準的な日中の勤務 |
| `late` | 遅出 | 遅い時間帯の勤務 |
| `night2` | 夜勤（二交代） | 16時間の夜間勤務 |
| `junnya` | 準夜（三交代） | 三交代制の準夜勤務 |
| `shinya` | 深夜（三交代） | 三交代制の深夜勤務 |
| `ake` | 明け | 二交代夜勤翌日の回復日 |
| `off` | 公休 | 通常の休日 |
| `paid` | 有給 | 有給休暇 |
| `refresh` | リフ休 | リフレッシュ休暇 |

## 2. 勤務区分（ワークタイプ）

| 区分コード | 区分名 | 使用可能な勤務 | ソルバー対象 |
|------------|--------|----------------|--------------|
| `2kohtai` | 二交代 | day, late, night2, ake, off, paid, refresh | 対象 |
| `3kohtai` | 三交代 | day, late, junnya, shinya, off, paid, refresh | 対象 |
| `day_only` | 日勤のみ | day, off, paid, refresh | 対象 |
| `fixed` | 固定パターン | パターンに依存 | **対象外** |

### 区分別の禁止勤務

- **二交代**: junnya, shinya は割当不可
- **三交代**: night2, ake は割当不可
- **日勤のみ**: night2, junnya, shinya, ake, late は割当不可

## 3. 夜勤の振り分けとローテーション

### 3.1 二交代制の夜勤ルール

#### 必須シーケンス（ハード制約）

```
night2 → ake → off（必ずこの順序で連続する）
```

- `night2` の翌日は **必ず** `ake`
- `ake` の翌日は **必ず** 休み（`off` / `paid` / `refresh` のいずれか）
- この3日間のシーケンスは絶対に崩せない

#### 月境界の処理

- 前月末が `night2` で終了 → 当月1日は `ake` から開始
- 前月末が `ake` で終了 → 当月1日は休み（`off` / `paid` / `refresh`）から開始

#### 二交代の間隔制約

| 間隔 | ペナルティ | 説明 |
|------|-----------|------|
| 2日間隔（night2...night2） | **30点** | 夜勤が近すぎる |
| 3日間隔 | **10点** | やや近い |

#### 日勤→夜勤の遷移

- `day` → `night2` の直接遷移: **40点ペナルティ**（休息不足）

---

### 3.2 三交代制の夜勤ルール

#### ハード制約（必須遵守）

1. **最大連続夜勤: 2日まで**
   - 3連続夜勤は禁止
   - 以下の2連続パターンは許可:
     - `shinya` → `shinya`（深夜→深夜）
     - `junnya` → `junnya`（準夜→準夜）
     - `shinya` → `junnya`（深夜→準夜）

2. **準夜の翌日制限**
   - `junnya` の翌日は `junnya` または `off` のみ
   - 日勤や深夜への遷移は不可

3. **準夜→深夜の遷移禁止**
   - `junnya` → `shinya` は不可

4. **深夜の翌日**（ソフト制約）
   - `shinya` の翌日は休み（off/paid/refresh）が**望ましい**
   - ハード制約ではなく、ペナルティ（5〜20点）で制御
   - 深夜→日勤など休みなしパターンも許容される（ただしペナルティ発生）

#### 三交代の月境界処理

- 前月末が `junnya` → 当月1日は `junnya` / `off` / `paid` / `refresh` のみ
- 前月末が `shinya` → 当月1日は休みが望ましい（ソフト制約、ペナルティで制御）

#### 三交代のペナルティ

> **重要**: 以下の「希望休み」にはリフレッシュ休暇（`refresh`）も含む。リフレッシュ休暇は希望休暇と同等の扱いとし、前日準夜・翌日深夜の回避ペナルティが同様に適用される。

| パターン | ペナルティ | 説明 |
|----------|-----------|------|
| junnya→shinya | **禁止** | 準夜から深夜への遷移（ハード制約） |
| shinya→休→shinya（散発夜勤、休=off/paid/refresh） | **15点** | 深夜が散発的 |
| junnya→休→shinya | **50点** | 準夜→休→深夜の体内リズム切替負担 |
| junnya→**希望休み/リフ休**→shinya | **100点** | 希望休み・リフ休前後での準夜→深夜（50+50の加重） |
| 希望休み/リフ休の前日が準夜 | **10点** | せっかくの休みの前夜が遅番勤務 |
| 希望休み/リフ休の翌日が深夜 | **10点** | 休み明けにいきなり深夜勤務 |
| shinya後に休みなし | **5〜20点** | 深夜後の休息不足 |
| day/late→shinya（翌日） | **25点** | 日勤帯→翌日深夜（インターバル約7時間、三交代専用） |

#### リフレッシュ休暇取得者の負担調整

リフレッシュ休暇を取得する職員は通常より多くの休みを得るため、他の日程できつめのローテーションを許容する。リフレッシュ日数に応じて以下のペナルティ・ボーナスを段階的に軽減し、リフレッシュを取っていない職員との公平性を確保する。

| リフレッシュ日数 | 深夜後非休 | 散発夜勤 | 準夜→休→深夜 | 好ましいローテーションボーナス |
|-----------------|-----------|---------|-------------|--------------------------|
| 0日（なし） | 20点 | 15点 | 50点 | -15点 |
| 1日 | 12点 | 10点 | 35点 | -10点 |
| 2日 | 7点 | 6点 | 25点 | -6点 |
| 3日以上 | 3点 | 3点 | 12点 | -3点 |

> リフレッシュ取得者はペナルティが軽減される（きつい配置を許容）と同時に、好ましいローテーションへのボーナスも減額される（優遇を控える）。希望休み前後の準夜・深夜回避ペナルティ（10点）はリフレッシュ日数に関わらず維持される。

#### 好ましいローテーション（ボーナス）

| パターン | ボーナス | 説明 |
|----------|---------|------|
| shinya→休→junnya | **-15〜-3点** | 深夜後に休んでから準夜（リフレッシュ日数で変動） |
| shinya→junnya→休 | **-15〜-3点** | 深夜→準夜と段階的にシフトダウン（リフレッシュ日数で変動） |

> ボーナスは目的関数から減算されるため、ソルバーがこれらのパターンを積極的に採用する。リフレッシュ取得者はボーナスが減額される。

#### 準夜・深夜のバランス

- 準夜と深夜の回数差: **10点/差分**
- 前半・後半の偏り: **15点/差分**

### 3.3 月間夜勤回数の上限

| 設定項目 | デフォルト値 | 説明 |
|----------|-------------|------|
| `maxNight` | **5回/月** | 1人あたりの最大夜勤回数 |

#### カウント方法

- **二交代**: `night2` の回数（各回2ポイントとして公平性計算）
- **三交代**: `junnya` + `shinya` の合計回数

#### 夜勤制限オプション

- `junnya_only`: 準夜のみ許可
- `shinya_only`: 深夜のみ許可

---

### 3.4 夜勤の公平分配

**目標**: 全スタッフ間で夜勤回数を均等化する

- **ペナルティ計算**: `(最大夜勤回数 − 最小夜勤回数) × 50点`
- ジニ係数・変動係数による公平性監視

## 4. 連続勤務制限

### ハード制約

```
最大連続勤務日数: 6日（7連勤禁止）
```

- 7日以上の連続勤務は禁止（ハード制約）
- `off`, `paid`, `ake`, `refresh` で連続がリセットされる
- **月境界**: 前月末の連続勤務日数を引き継いで判定

### ソフト制約

- 5日連続勤務: **150点ペナルティ**
- 6日連続勤務: **300点ペナルティ**（必要な場合は許容されるが強く抑制）

## 5. 月間公休日数

### 基本日数（全病棟共通）

| 月 | 公休日数 | 備考 |
|----|---------|------|
| 2月 | **8日** | 短い月 |
| 5月 | **10日** | GW考慮 |
| その他 | **9日** | 標準 |

### 公休日数の制約

- **`off`（公休）のみ** が monthlyOff と一致する必要がある（ハード制約）
- `paid`（有給）と `refresh`（リフ休）は公休に**追加**される休み（取得すると休み日数が増える）
- 有給は本人の希望日のみに割当可能
- リフレッシュは希望日に割当。他の制約との兼ね合いで緩和される場合がある

## 6. 日別必要人数

### 日祝の人員配置ルール

日曜・祝日の日勤は **最低限 ～ 最低限+1名** の範囲に制約され、過剰配置を防止する。準夜・深夜は曜日に関係なく月を通して固定人数。

### 6.1 日勤帯

| 条件 | 必要人数（デフォルト） | 上限 |
|------|---------------------|------|
| 平日 | **7名**（下限のみ） | なし |

| 曜日 | 下限目標 | 備考 |
|------|---------|------|
| 月・木 | 基準値以上 | |
| 水・金 | 基準値−1以上 | |
| 火・土 | 基準値−1 or −2以上 | 下限は基準値−2 |
| 日・祝 | 休日基準値〜休日基準値+1 | 上限あり |

> 日勤人数の不足はソフト制約（500点/人ペナルティ）。必要な場合は目標未満も許容される。

#### 二病棟の曜日別調整（ソフト制約）

| 曜日 | 下限目標 | 備考 |
|------|---------|------|
| 月・金 | 基準値以上 | |

### 6.2 準夜帯（夕方〜深夜前）

### 6.3 深夜帯（深夜〜早朝）

## 8. 希望シフト（Wish）の扱い

### 希望の種類

| 種類 | 説明 | ペナルティ |
|------|------|-----------|
| `assign`（指定） | 特定日に特定勤務を希望 | **1000点/違反** |
| `avoid`（回避） | 特定日に特定勤務を回避希望 | **1000点/違反** |

### 特殊ルール

- **希望休み（off の assign）**: **ハード制約**として必ず遵守（緩和不可）
- **リフレッシュ休暇（refresh）**: 希望日に割当。公休数には影響しない（追加休み）。前日準夜・翌日深夜の回避ペナルティ適用。解なし時は緩和可能
- **有給（paid）**: 本人の希望がある日のみ割当可能。公休数には影響しない（追加休み）
- 解が見つからない場合、希望休み以外の wish は緩和される場合がある

## 9. 公平性指標とペナルティ体系

### ペナルティ一覧（重み順）

| 制約 | ペナルティ | 種類 |
|------|-----------|------|
| 希望違反（assign/avoid） | **1000点** | ソフト |
| 人員不足 | **500点/人** | ソフト |
| 5日連続勤務 | **150点** | ソフト |
| 夜勤回数偏り | **50点/差分** | ソフト |
| 日勤→夜勤遷移 | **40点** | ソフト |
| 土日勤務偏り | **40点/差分** | ソフト |
| 夜勤2日間隔 | **30点** | ソフト |
| 遅出回数偏り | **20点** | ソフト |
| 三交代連続夜勤 | **20点** | ソフト |
| 準夜→休→深夜パターン | **50点** | ソフト |
| 準夜→希望休み/リフ休→深夜 | **100点** | ソフト |
| 散発夜勤 | **15点** | ソフト |
| 準夜・深夜半月偏り | **15点** | ソフト |
| 夜勤3日間隔 | **10点** | ソフト |
| 準夜・深夜バランス | **10点** | ソフト |
| 深夜後非休 | **5〜20点** | ソフト |

### 公平性の測定

| 指標 | 算出方法 | 基準 |
|------|---------|------|
| ジニ係数 | 0（完全平等）〜 1（完全不平等） | < 0.1: 優秀 / 0.1-0.2: 良好 / 0.2-0.3: 普通 / > 0.3: 要改善 |
| 変動係数（CV） | 標準偏差 ÷ 平均 | 低いほど良い |
| 夜勤レンジ | 最大回数 − 最小回数 | 0に近いほど良い |

## 10. 祝日の扱い

- **データソース**: `holidays.json` に定義
- **扱い**: 土日と同等（休日用の人員配置を適用）
- **日勤人数**: 休日基準値（デフォルト5名）を使用
- **公平性**: 土日勤務カウントに祝日出勤も含む

## 11. ソルバーの最適化戦略

### 解法モード

| モード | 制限時間 | 用途 |
|--------|---------|------|
| `quick` | **15秒** | 高速な仮スケジュール |
| `balanced` | **45秒** | 通常使用 |
| `quality` | **90秒** | 最高品質 |

### フォールバック戦略

現在のソルバーは **厳密解法のみ**（カスケード廃止）。`_solve_core(attempt_level=1)` を1回だけ実行し、解が見つからない場合はエラーを返す。公休日数 (`monthlyOff`) は常に `==` ハード制約で厳守。

> **注意**: `_solve_core()` には `attempt_level` 引数（L2〜L5+）による制約緩和コードが残存しているが、`solve()` メソッドは常に `attempt_level=1` で呼び出すため、これらは現在デッドコードである。

### 目的関数（最小化対象）

```
最小化:
  希望・人員違反の合計
  + 夜勤レンジ段階ペナルティ               ... 夜勤公平性（レンジが大きいほど急増）
    レンジ1: 50pt, 2: 150pt, 3: 350pt, 4: 700pt, 5: 1200pt
  + 土日勤務偏り × 40                     ... 土日公平性
  + 夜勤間隔ペナルティ                     ... 夜勤の間隔
  + 準夜・深夜バランスペナルティ            ... 三交代バランス
  + 遅出偏り × 20                         ... 遅出公平性
  + 散発夜勤ペナルティ                     ... 夜勤の散発防止
  + 深夜後非休ペナルティ                   ... 深夜後の休息
  （二交代の日勤→夜勤ペナルティは廃止: インターバル20時間以上で問題なし）
  + 日勤帯→深夜ペナルティ × 25           ... 三交代専用（インターバル約7時間）
  + 準夜→休→深夜ペナルティ               ... 体内リズム切替負担
  + 希望休み前後夜勤ペナルティ             ... 希望休みの質の確保
  - 好ましいローテーションボーナス         ... 深夜→休→準夜、深夜→準夜→休を優遇
  + 連続勤務ペナルティ                     ... 連続日数抑制
```
