## JSON構造

各JSONファイルは以下の共通構造を持つ:

```
documentType   : 書類名
version        : 基準のバージョン
sourceDocuments: 原典ファイル名
fields[]       : フィールド定義の配列
  id           : フィールドID
  name         : 様式上の項目名
  criticality  : critical（最重要）/ high（重要）/ normal（通常）
  inputType    : textarea / text / checklist / date 等
  description  : 審査ポイントの要約
  rules[]      : チェックルールの配列
    id           : ルールID
    description  : チェック内容
    details      : 詳細説明（任意）
    badExamples  : 不十分な記載例（任意）
    goodExamples : 望ましい記載例（任意）
    returnReason : 返戻となる条件（任意）
crossFieldRules[]: フィールド間の整合性チェック
```

## 基準更新時の編集ガイド

1. 該当するJSONファイルを開く
2. `fields[].rules[]` にルールを追加・修正・削除
3. `badExamples` / `goodExamples` を更新（AIチェック精度に直結）
4. `version` を更新
5. JSONの構文チェック: `python -c "import json; json.load(open('ファイル名', encoding='utf-8')); print('OK')"`

## AIチェック機能への転用（将来）

これらのJSONファイルは、Claude APIを使ったAIチェック機能のシステムプロンプトの素材として直接利用できる設計になっている:

1. バックエンドがJSONを読み込み、フィールドごとのルール・例をシステムプロンプトに埋め込む
2. 医師が入力した書類の各フィールド内容をユーザーメッセージとして送信
3. AIがフィールドごとにpass/warning/failを判定し、具体的なフィードバックを返す
